Features

像在本地一样用 CC，无论你在哪

RCC 不是一个新的 AI Agent。它只做一件事：把你本地那个真实的 Claude Code 进程，安全地搬到任意浏览器里。

⚡

不改 CC 任何一行

Host daemon 用 pty 包住真实的 claude 命令。 所有 slash commands / hooks / MCP / plan mode 原样生效。CC 官方升级，RCC 自动受益。

🌐

出站隧道，零配置穿透

基于 Cloudflare Tunnel。无需路由器映射、DDNS、VPN。 第一次启动自动给你一个固定子域：rcc-yourname.trycf.com。

📱

手机是一等公民

不是把桌面版塞进小屏。语义化卡片、虚拟键盘上方的 CC 快捷键条、 长按说话、权限请求直推锁屏。地铁上也能推进任务。

🔁

多端镜像，无缝接续

电脑上开的任务，手机打开立刻看到进度。CRDT 同步输入状态， 两台设备同时写字也不会冲突。断线自动重连补帧，不丢字符。

🛡

权限审批走移动端

CC 要执行危险命令？弹窗推到你手机锁屏。Face ID 一按即允许， 未确认不会执行。比坐在电脑前更安全。

🔐

端到端加密

Passkey (WebAuthn) 免密码登录。应用层 libsodium E2E， 即便 Cloudflare 被攻破，攻击者只看到密文。

How it works

架构一图看懂

你的 Mac 主动打一条出站连接到 Cloudflare 边缘节点。浏览器从另一头拨进来， 中间只做转发、不解密。你的代码从不离开本机。

Use cases

它什么时候最有用

🚇

通勤路上继续推进

出门前让 CC 开始重构，地铁上打开手机看进度、批准权限、发新指令。 到公司前代码已经提交好 PR。

✈

出差不用背电脑

家里的 Mac 保持开机，带个 iPad 出差。在酒店 WiFi 下照常跑 CC， 享受家里机器的算力、MCP 配置、本地数据库。

🛋

沙发上写代码

不想回书房？拿着手机语音下指令："把 Header 改成 sticky"。 CC 改完、diff 推过来、你点个允许。整个过程不用坐起来。

Security

比坐在电脑前还安全

传统 SSH 远程方案有"完整 shell 权限"的问题。RCC 针对 AI Agent 场景重新设计了权限模型： 所有破坏性操作都需要移动端 Face ID 二次确认，形成天然的"双钥匙"。

🔐
无密码 · Passkey
WebAuthn 标准。私钥在手机 Secure Enclave 里，连你自己都导不出来。
📦
应用层 E2E
libsodium 在进入 TLS 前再加一层。Cloudflare 只看到加密字节。
🚫
设备管理 · 一键吊销
手机丢了？从电脑端一条命令撤销它的 passkey，该设备立即断线。
⏸
默认 Plan Mode
远程连接时 CC 默认不执行任何写操作，除非你明确同意。
# threat model
[×] MITM eavesdropping → TLS + E2E
[×] CF account pwned → sees ciphertext only
[×] Stolen phone → revoke device
[×] Replay attacks → nonce + sliding window
[×] Malicious prompt → rm -rf → Face ID approve
[×] Unknown device → passkey required
# known residual risks
[!] Host Mac compromised → out of scope
[!] You approve a bad command → read before tapping allow
